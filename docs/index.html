Business Background
业务背景
企业协作 SaaS 多租户平台，支持 Meeting（会议）、Calling（通话）、Workspace（工作空间）、 Messaging（消息）等核心协作模块。Supervisor 角色可查看所在公司（org_id）的运营 Dashboard， 实时掌握团队协作效率。
🎥

多租户隔离

基于 org_id 强制隔离，Supervisor 只能查询本公司数据，杜绝跨组织数据泄露。

🗄️

生产级数据源

实际业务已接入 Apache Pinot（实时分析）、Trino（跨源查询）、TiDB（HTAP）；学习环境使用 PostgreSQL，保留完整 SQL 语法和数据模型设计思路。

主管可看到所在公司的会议质量、通话时长、信息发送数、设备使用率、使用时间、会议平均时长等运营指标。

Core Advantages
核心优势
6 大核心技术优势，覆盖 NL2SQL 全链路工程实践
01
🔍

三路 RAG 检索

Schema 表/字段双粒度 chunk + 25+ 业务指标库（含别名映射）+ 15+ SQL 示例，三路并行向量检索（Milvus COSINE），为 LLM 提供完整上下文，大幅降低幻觉。

02
⚡

LangGraph 状态机

6 个可观测节点 + 条件路由 + 失败重试，意图识别 → RAG → 生成 → 校验 → 执行 → 图表推荐，每一步都可独立调试和替换 Prompt。

03
🏢

多租户 org_id 隔离

所有 SQL 强制注入 WHERE org_id = ?，RAG 检索结果按 org_id 过滤，Supervisor 只能看到本公司数据，满足企业级数据安全要求。

04
📈

智能图表推荐

LLM 决策 + 规则兜底双保险。根据时间序列/维度基数/数值列数自动选型（line/bar/pie/area/table），无需用户手动指定。

05
💾

Milvus Lite 零运维

纯 Python 包，pip install 即可，无需 Docker，无需额外服务进程。Milvus Lite 数据存储在本地文件，数据目录 ./data/milvus/。

06
🔄

流式 SSE 输出

Server-Sent Events 实时推送 Agent 思考过程，前端逐步展示意图识别→Schema 检索→SQL 生成→执行的完整链路，体验接近 ChatGPT。

Week 5-6

Quick Start
快速启动
5 步完成全量启动（依赖安装 → 数据生成 → 索引构建 → 启动服务）

方式一：一键脚本（推荐）

$ bash scripts/start.sh
# 自动完成：venv → pip install → PostgreSQL 建表 → 生成 ~3000 条数据 → 构建向量索引

方式二：手动启动

# 1. 创建虚拟环境
$ python3 -m venv .venv && source .venv/bin/activate
# 2. 安装依赖
$ pip install -r requirements.txt
# 3. 生成模拟数据
$ python -m scripts.generate_mock_data
# 4. 构建向量索引
$ python -c "from backend.rag.retriever import *; UnifiedRetriever().build_all_indexes(drop_existing=True)"
# 5. 启动后端（端口 8000）
$ uvicorn backend.main:app --reload --port 8000
# 6. 启动前端（新终端）
$ cd frontend && npm install && npm run dev

Example Queries
支持的查询类型
Supervisor 输入自然语言问题，系统自动推断意图、检索 Schema、生成 SQL 并渲染图表
📹 会议
"本周各部门开了多少次会？"
→ GROUP BY department, COUNT(*)
📞 通话
"最近一个月通话接通率是多少？"
→ completed / total COUNT
💼 工作空间
"各部门用户平均使用时长是多少？"
→ AVG(active_time_seconds)
💬 消息
"每天消息发送量趋势如何？"
→ DATE × COUNT, 折线图
📡 质量
"各部门会议质量评分对比"
→ AVG(quality_score) × department
💻 设备
"设备使用分布（桌面端/移动端）"
→ GROUP BY device_type, 饼图
